🏗 1️⃣ FINAL DATABASE DESIGN (PRODUCTION READY)
🔐 1. USER TABLE (Custom User Model)

We extend AbstractUser.

Table: users
Field	Type	Purpose
id	PK	User ID
username	CharField	Login
email	EmailField	Unique
password	Hashed	Secure
phone_number	CharField	Customer contact
role	ChoiceField	admin / customer
is_active	Boolean	Soft delete
date_joined	DateTime	Auto
Why?

One table for admin + customer

Role-based permissions

JWT integration ready

📂 2. CATEGORY TABLE
Table: categories
Field	Type
id	PK
name	CharField
description	TextField
is_active	Boolean
created_at	DateTime
updated_at	DateTime

Relationship:

Category → One-to-Many → Product
🥛 3. PRODUCT TABLE
Table: products
Field	Type
id	PK
name	CharField
category	FK(Category)
description	TextField
image	ImageField
one_time_price	Decimal
monthly_price	Decimal
quarterly_price	Decimal
yearly_price	Decimal
stock_quantity	Integer
is_active	Boolean
created_at	DateTime
updated_at	DateTime

Relationship:

Product → belongs to Category
Product → used in CartItem
Product → used in OrderItem
🛒 4. CART TABLE
Table: carts
Field	Type
id	PK
user	FK(User)
created_at	DateTime
updated_at	DateTime

One active cart per user.

🛒 5. CART ITEM TABLE
Table: cart_items
Field	Type
id	PK
cart	FK(Cart)
product	FK(Product)
quantity	Integer
plan_type	ChoiceField (one_time, monthly, quarterly, yearly)
price_at_time	Decimal
created_at	DateTime

Why store price_at_time?
→ If product price changes tomorrow, cart still has old price.

📦 6. ORDER TABLE
Table: orders
Field	Type
id	PK
user	FK(User)
total_amount	Decimal
status	ChoiceField
payment_status	ChoiceField
shipping_address	TextField
created_at	DateTime
updated_at	DateTime

Order Status:

pending

confirmed

shipped

delivered

cancelled

Payment Status:

unpaid

paid

failed

📦 7. ORDER ITEM TABLE
Table: order_items
Field	Type
id	PK
order	FK(Order)
product	FK(Product)
quantity	Integer
plan_type	ChoiceField
price_at_purchase	Decimal

Cart → Converts → Order + OrderItems

💳 8. PAYMENT TABLE
Table: payments
Field	Type

status	ChoiceField

🔄 COMPLETE DATA FLOW
🟢 1. User Registration

User → Register → JWT Token issued

🟢 2. Browse Products

User:

GET /products

Pagination applied

Filtering by category

Search by name

🟢 3. Add to Cart

User:
POST /cart/add/

Creates:

Cart (if not exists)

CartItem

🟢 4. Checkout

User:
POST /orders/create/

Flow:

Read cart items

Calculate total

Create Order

Create OrderItems

Clear cart

🟢 5. Payment

User:
POST /payment/

Payment created

If success → order.payment_status = paid

🔍 PAGINATION (DRF)

In settings.py:

REST_FRAMEWORK = {
    "DEFAULT_PAGINATION_CLASS": "rest_framework.pagination.PageNumberPagination",
    "PAGE_SIZE": 10,
}

Example:

GET /products/?page=2

    "rest_framework.filters.OrderingFilter",
]

Example:
